# asset/ - pack-authorable custom asset types (the asset-driven config layer)

Router for `asset/`. Kweebec's tunable content is **asset-driven** (Pattern A: a structured `AssetBuilderCodec` per type, mirroring hyMMO's `asset/type/` + `AssetStoreRegistrar`), pack-overridable AND runtime-overridable. **PARADIGM: a new tunable/content type is a pack-authorable custom asset type via a codec - never hardcoded.** The whole layer is registered in `KweebecNightmarePlugin.setup()` via [`KweebecAssetRegistrar`](KweebecAssetRegistrar.java).

- **[`RoundPresetAsset`](RoundPresetAsset.java)** - a difficulty preset (`Server/KweebecNightmare/Presets/*.json`; PascalCase filenames). A flat structured codec of the 13 `RuleSet` knobs + `NameKey`/`Enabled`/`InventoryMode`/`RewardOnExit`/`HunterArchetype`; `toRuleSet(id)` builds the unchanged runtime `RuleSet`. **Replaces the old hardcoded `RoundPreset` enum as the schema authority** ([`RoundPreset`](../round/RoundPreset.java) is now a thin shim delegating to `PresetConfig`).
- **[`PresetConfig`](PresetConfig.java)** - the fold authority (`defaults < pack < owner`): jar baselines from [`DefaultPresets`](DefaultPresets.java) (Amateur/Nightmare/Hardcore, matching the old enum exactly), `mergePackLayer` overlays a pack's decoded presets (id lowercased so they overlay the lowercase defaults), `resolve(id)` returns a `RuleSet`. The 4th **runtime** tier sits above this in [`../integration/KweebecNightmareAPI`](../integration/KweebecNightmareAPI.java).
- **[`HunterArchetypeAsset`](HunterArchetypeAsset.java)** + **[`HunterArchetypeConfig`](HunterArchetypeConfig.java)** + **[`DefaultHunters`](DefaultHunters.java)** - hunter archetype data (`Server/KweebecNightmare/Hunters/*.json`: RoleName/Count/Weight/SpeedBands/BandEffectIds/SpawnTier/Kind). Authored AHEAD of its consumer (the shipped `Stalker.json` reproduces today's single hunter); **Phase 2A** generalizes `AiHunterController` to read this roster (Stalker/Lunger/Spitter/Ambusher) - do NOT hardcode-then-refactor.
- **[`KweebecAssetRegistrar`](KweebecAssetRegistrar.java)** - mirrors hyMMO's `AssetStoreRegistrar` field-for-field: `registerStore` (the `HytaleAssetStore.Builder` chain) for Control + Presets + Hunters, both content stores `loadsAfter(KweebecPackControlAsset.class)`, and a `LoadedAssetsEvent` typed-map fold per type (the Pattern-A `onQuestGiverAssetsLoaded` shape). [`KweebecPackControlAsset`](KweebecPackControlAsset.java) + `KweebecPackControlRegistry` resolve per-type add/replace.
- **Gotchas (shipped bugs):** asset FILENAMES must be PascalCase (`Stalker.json`, not `stalker.json` - lowercase logs `Asset key ... has incorrect format!`); codec field keys start Upper-case (the codec constructor throws at static init otherwise). Use `util.SafeLog` (not the raw flogger LOGGER) on these load/parse paths.
