# asset/ - pack-authorable custom asset types (the asset-driven config layer)

Router for `asset/`. Kweebec's tunable content is **asset-driven** (Pattern A: a structured `AssetBuilderCodec` per type, mirroring hyMMO's `asset/type/` + `AssetStoreRegistrar`), pack-overridable AND runtime-overridable. **PARADIGM: a new tunable/content type is a pack-authorable custom asset type via a codec - never hardcoded.** The whole layer is registered in `KweebecNightmarePlugin.setup()` via [`KweebecAssetRegistrar`](KweebecAssetRegistrar.java).

> **MIGRATED to ziggfreed-common (asset-driven-framework pass).** Four encounter types moved OUT of this package into the reusable common framework + their content into `Server/ZiggfreedCommon/<Type>/`: **Boss** (`BossAsset`/`BossConfig`/`DefaultBosses` -> `common.instance.encounter.MultiPhaseBoss{Asset,Config}`, content `ZiggfreedCommon/Bosses/`), **ScareBeat** (-> `common.instance.effect.BandedEffect{Asset,Config}`, content `ZiggfreedCommon/BandedEffects/`, `MinCorruptionTier` -> `MinTier`), **SpawnRule** (-> `common.instance.encounter.EncounterRule{Asset,Config}`, content `ZiggfreedCommon/EncounterRules/`; kweebec's `Trigger`/`Placement` enums live in [`../hunter/SpawnTrigger`](../hunter/SpawnTrigger.java) + [`SpawnPlacementKind`](../hunter/SpawnPlacementKind.java) and parse the common asset's free-string getters), **Structure** (-> `common.world.WeightedPrefabPlacement{Asset,Config}`, content `ZiggfreedCommon/PrefabPlacements/`). Also the cross-cutting **InstancePreset** store is common-owned now (`ZiggfreedCommon/Instances/`). The kweebec consumers (`BossController`/`ScareDirector`/`AiHunterController`/`StructureCatalog`) READ the common config singletons; the runtime-override seams stay on `KweebecNightmareAPI`. The bullets below cover ONLY the types still kweebec-owned: **RoundPreset, HunterArchetype, Mutator** (+ Control). See [[ziggfreed-common-asset-driven-configs]].

- **[`RoundPresetAsset`](RoundPresetAsset.java)** - a difficulty preset (`Server/KweebecNightmare/Presets/*.json`; PascalCase filenames). A flat structured codec of the `RuleSet` knobs + `InventoryMode`/`HunterArchetype` (the cross-cutting `NameKey`/`Enabled`/`RewardOnExit` now live ONLY on the co-keyed common `InstancePresetAsset` at `Server/ZiggfreedCommon/Instances/`) + the **8 asset-driven scoring knobs** (`Baseline`/`ParTimeSeconds`/`TimePointsPerSecond`/`DamagePointsPerHp`/`DamageBudget`/`StunBonusPer`/`ShrineBonusPer`/`AllShrinesBonus`); `toRuleSet(id)` builds the runtime `RuleSet`, folding the scoring knobs into [`RuleSet.scoring()`](../round/RuleSet.java) (a per-preset [`ScoringConfig`](../score/ScoringConfig.java); absent = the ScoringConfig default). The **shrine-discovery knobs** `ShrineDiscovery` (`OFF`/`ON_INTERACT`/`PROXIMITY`) / `ShrineDiscoveryVisibility` (`SELF`/`ALL` aliases, or `PER_PLAYER`/`SHARED`) / `ShrineDiscoveryRadius` drive the world-map shrine markers (ziggfreed-common `MapDiscovery`); Amateur authors `ALL`, Hardcore `OFF`. So scoring is per-difficulty + pack-authorable; the runtime tier ([`../integration/KweebecNightmareAPI`](../integration/KweebecNightmareAPI.java)`.resolveScoring(base)`) can still override/scale. **Replaces the old hardcoded `RoundPreset` enum as the schema authority** ([`RoundPreset`](../round/RoundPreset.java) is now a thin shim delegating to `PresetConfig`).
- **[`PresetConfig`](PresetConfig.java)** - the fold authority (`defaults < pack < owner`): jar baselines from [`DefaultPresets`](DefaultPresets.java) (Amateur/Nightmare/Hardcore, matching the old enum exactly), `mergePackLayer` overlays a pack's decoded presets (id lowercased so they overlay the lowercase defaults), `resolve(id)` returns a `RuleSet`. The 4th **runtime** tier sits above this in [`../integration/KweebecNightmareAPI`](../integration/KweebecNightmareAPI.java).
- **[`HunterArchetypeAsset`](HunterArchetypeAsset.java)** + **[`HunterArchetypeConfig`](HunterArchetypeConfig.java)** + **[`DefaultHunters`](DefaultHunters.java)** - hunter archetype data (`Server/KweebecNightmare/Hunters/*.json`: RoleName/Count/Weight/SpeedBands/BandEffectIds/SpawnTier/Kind). As of 0.3.0 `AiHunterController` CONSUMES this as a weighted per-hunter roster (Stalker/Lunger/Spitter/Ambusher; the latter three ship as `Lunger.json`/`Spitter.json`/`Ambusher.json` + their Variant roles).
- **[`MutatorAsset`](MutatorAsset.java)** + **[`MutatorConfig`](MutatorConfig.java)** + **[`DefaultMutators`](DefaultMutators.java)** (0.3.0) - a stackable round modifier (`Server/KweebecNightmare/Mutators/*.json`: additive deltas HunterCountDelta/HunterSpeedMaxDelta/CaveShrineCountDelta/CorruptionPerSecondDelta/ShrineRelightSecondsDelta/RoundCapSecondsDelta/ShrineBaseDelta). A preset names mutators via `RoundPresetAsset.Mutators`; `PresetConfig.resolve` applies them (additive, commutative) between the defaults<pack<owner fold and the runtime SCALE tier. Jar set: Swarm/Relentless/Caverns/Decay/Blitz.
- **[`GroveThrowableAsset`](GroveThrowableAsset.java)** + **[`GroveThrowableConfig`](GroveThrowableConfig.java)** (0.6.0) - a harvestable glow-throwable the grove grows (`Server/KweebecNightmare/GroveThrowables/*.json`: Enabled/PrefabKey/PerShrineCount/ScatterCount/RespawnWithWaves/MinCorruptionTier), the data-driven generalization of the hardcoded Moonbloom scatter for the utility variants (Gustbloom knockback, Mirebloom slow) + any future one. ZERO jar defaults (pure pack content); ships the Gust/Mire entries `Enabled:false` so the gather loop is present but DORMANT. `ArenaBuilder.plantGroveThrowables` iterates `GroveThrowableConfig.placeable()` (the generalized `plantClusters` over each entry's `PrefabKey`), driven for the initial supply from `ShrinePlacement` + each Moonbloom wave from `ChaseMode`. ADD-only (no Control key), like Mutators. Add a variant = a new JSON + its asset chain ([`../moonbloom/GlowThrowables`](../moonbloom/GlowThrowables.java) holds the ids). The boss-phase Emberbloom (damage) is NOT a grove throwable - its clusters are placed by `BossController` from the common `MultiPhaseBossAsset` `ThrowableClusterId`/`PhaseNThrowableCount`/`ThrowableRingRadius`/`ThrowableRespawnSeconds` knobs.
- **(Structure / ScareBeat / SpawnRule / Boss MIGRATED to ziggfreed-common - see the banner above.)** Their content authors at `Server/ZiggfreedCommon/PrefabPlacements|BandedEffects|EncounterRules|Bosses/`; the consumers read `common.world.WeightedPrefabPlacementConfig` / `common.instance.effect.BandedEffectConfig` / `common.instance.encounter.EncounterRuleConfig` / `common.instance.encounter.MultiPhaseBossConfig`.
- **[`KweebecAssetRegistrar`](KweebecAssetRegistrar.java)** - routes through ziggfreed-common's `AssetStoreRegistrar.registerStore` (the byte-identical private copy was deleted): `registerStore` for Control + **Presets + Hunters + Mutators** (the remaining kweebec-owned types; all `loadsAfter(KweebecPackControlAsset.class)`), and a `LoadedAssetsEvent` typed-map fold per type. The migrated types' stores + the cross-cutting InstancePreset store are registered by common's `FrameworkAssetRegistrar`, NOT here. [`KweebecPackControlAsset`](KweebecPackControlAsset.java) (now just `Presets`/`Hunters`) + `KweebecPackControlRegistry` resolve per-type add/replace.
- **Gotchas (shipped bugs):** asset FILENAMES must be PascalCase (`Stalker.json`, not `stalker.json` - lowercase logs `Asset key ... has incorrect format!`); codec field keys start Upper-case (the codec constructor throws at static init otherwise). Use `util.SafeLog` (not the raw flogger LOGGER) on these load/parse paths.
