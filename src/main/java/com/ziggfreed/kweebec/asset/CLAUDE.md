# asset/ - pack-authorable custom asset types (the asset-driven config layer)

Router for `asset/`. Kweebec's tunable content is **asset-driven** (Pattern A: a structured `AssetBuilderCodec` per type, mirroring hyMMO's `asset/type/` + `AssetStoreRegistrar`), pack-overridable AND runtime-overridable. **PARADIGM: a new tunable/content type is a pack-authorable custom asset type via a codec - never hardcoded.** The whole layer is registered in `KweebecNightmarePlugin.setup()` via [`KweebecAssetRegistrar`](KweebecAssetRegistrar.java).

- **[`RoundPresetAsset`](RoundPresetAsset.java)** - a difficulty preset (`Server/KweebecNightmare/Presets/*.json`; PascalCase filenames). A flat structured codec of the `RuleSet` knobs + `NameKey`/`Enabled`/`InventoryMode`/`RewardOnExit`/`HunterArchetype` + the **8 asset-driven scoring knobs** (`Baseline`/`ParTimeSeconds`/`TimePointsPerSecond`/`DamagePointsPerHp`/`DamageBudget`/`StunBonusPer`/`ShrineBonusPer`/`AllShrinesBonus`); `toRuleSet(id)` builds the runtime `RuleSet`, folding the scoring knobs into [`RuleSet.scoring()`](../round/RuleSet.java) (a per-preset [`ScoringConfig`](../score/ScoringConfig.java); absent = the ScoringConfig default). So scoring is per-difficulty + pack-authorable; the runtime tier ([`../integration/KweebecNightmareAPI`](../integration/KweebecNightmareAPI.java)`.resolveScoring(base)`) can still override/scale. **Replaces the old hardcoded `RoundPreset` enum as the schema authority** ([`RoundPreset`](../round/RoundPreset.java) is now a thin shim delegating to `PresetConfig`).
- **[`PresetConfig`](PresetConfig.java)** - the fold authority (`defaults < pack < owner`): jar baselines from [`DefaultPresets`](DefaultPresets.java) (Amateur/Nightmare/Hardcore, matching the old enum exactly), `mergePackLayer` overlays a pack's decoded presets (id lowercased so they overlay the lowercase defaults), `resolve(id)` returns a `RuleSet`. The 4th **runtime** tier sits above this in [`../integration/KweebecNightmareAPI`](../integration/KweebecNightmareAPI.java).
- **[`HunterArchetypeAsset`](HunterArchetypeAsset.java)** + **[`HunterArchetypeConfig`](HunterArchetypeConfig.java)** + **[`DefaultHunters`](DefaultHunters.java)** - hunter archetype data (`Server/KweebecNightmare/Hunters/*.json`: RoleName/Count/Weight/SpeedBands/BandEffectIds/SpawnTier/Kind). As of 0.3.0 `AiHunterController` CONSUMES this as a weighted per-hunter roster (Stalker/Lunger/Spitter/Ambusher; the latter three ship as `Lunger.json`/`Spitter.json`/`Ambusher.json` + their Variant roles).
- **[`MutatorAsset`](MutatorAsset.java)** + **[`MutatorConfig`](MutatorConfig.java)** + **[`DefaultMutators`](DefaultMutators.java)** (0.3.0) - a stackable round modifier (`Server/KweebecNightmare/Mutators/*.json`: additive deltas HunterCountDelta/HunterSpeedMaxDelta/CaveShrineCountDelta/CorruptionPerSecondDelta/ShrineRelightSecondsDelta/RoundCapSecondsDelta/ShrineBaseDelta). A preset names mutators via `RoundPresetAsset.Mutators`; `PresetConfig.resolve` applies them (additive, commutative) between the defaults<pack<owner fold and the runtime SCALE tier. Jar set: Swarm/Relentless/Caverns/Decay/Blitz.
- **[`StructurePlacementAsset`](StructurePlacementAsset.java)** + **[`StructureConfig`](StructureConfig.java)** + **[`DefaultStructures`](DefaultStructures.java)** (0.3.0) - the corrupted-structure placement table (`Server/KweebecNightmare/Structures/*.json`: PrefabKey/Role/X/Z/Weight, one placement per file). [`../arena/StructureCatalog`](../arena/StructureCatalog.java) reads `StructureConfig.all()` for its seeded per-round selection (it keeps only the selection logic; the data is asset-driven).
- **[`ScareBeatAsset`](ScareBeatAsset.java)** + **[`ScareBeatConfig`](ScareBeatConfig.java)** + **[`DefaultScareBeats`](DefaultScareBeats.java)** (0.3.0) - the tier/proximity -> EntityEffect mapping (`Server/KweebecNightmare/ScareBeats/*.json`: Band/MinCorruptionTier/EffectId/SoundId/OneShot) the [`../feedback/ScareDirector`](../feedback/ScareDirector.java) fans (NO hardcoded mapping). Bands -> `KweebecNightmare_Dread_1/2/3`; the jumpscare beat -> `KweebecNightmare_Jumpscare` + `SFX_Hedera_Scream`.
- **[`KweebecAssetRegistrar`](KweebecAssetRegistrar.java)** - mirrors hyMMO's `AssetStoreRegistrar` field-for-field: `registerStore` for Control + Presets + Hunters + **Mutators + Structures + ScareBeats** (all content stores `loadsAfter(KweebecPackControlAsset.class)`), and a `LoadedAssetsEvent` typed-map fold per type. The three 0.3.0 stores are ADD-only (replace=false) until `KweebecPackControlAsset` grows their control keys. [`KweebecPackControlAsset`](KweebecPackControlAsset.java) + `KweebecPackControlRegistry` resolve per-type add/replace.
- **Gotchas (shipped bugs):** asset FILENAMES must be PascalCase (`Stalker.json`, not `stalker.json` - lowercase logs `Asset key ... has incorrect format!`); codec field keys start Upper-case (the codec constructor throws at static init otherwise). Use `util.SafeLog` (not the raw flogger LOGGER) on these load/parse paths.
