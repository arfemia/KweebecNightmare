# arena/ - the Chase grove skeleton (anchors + prefab stamping)

Router for `arena/`. The mod-root `CLAUDE.md` (loads with the kweebec subtree) has the Path-A-instance overview; the kweebec techspec (linked from the mod root) has the verified `PrefabUtil.paste` signature. Gameplay runs ENTIRELY off the anchors here, so a round is playable before any prefab pastes - the geometry is a visual layer, never a logic dependency.

- **[`ArenaLayout`](ArenaLayout.java) is the fixed coordinate authority** - `SPAWN`/`GATE`/`ESCAPE`/`HUNTER_DEN` plus the shrine ring (`shrineAnchors(count)`, deterministic so every client agrees on the layout) AND `SHRINE_CAVE` (the one sub-surface objective). Surface anchors sit at `STAND_Y`; `SHRINE_CAVE` sits at `CAVE_STAND_Y` (the chamber stand level); `count` comes from `RuleSet.shrineCount(partySize)`. Proximity gates use `INTERACT_RADIUS_SQ` (shrine/rescue) and `ESCAPE_RADIUS_SQ`.
- **The floor Y MUST match the worldgen, or players spawn inside terrain.** The Chase instance worldgen is the pack-authored `KweebecNightmare_Grove` (a Default_Flat structural clone, WorldStructure `Base` = 80) whose biome density is a **flat play disc out to r~70** (the whole gameplay footprint), then rolling hills (r70-110), then a finite cliff into the void (~r112). Inside the disc the walkable surface top is world Y=80 (`STAND_Y`) over a top solid block at Y=79 (`FLOOR_Y`); `instance.bson` `SpawnProvider.SpawnPoint.Y` MUST agree (80). The original Y=65 buried players ~15 blocks underground (the 0.2.0 bug). **Every surface anchor stays inside r70 so the flat-disc contract holds** (lead with the radial-mask flat disc, NOT heightmap re-derivation; the disc is byte-for-byte flat because the hill/cave/edge density terms multiply to 0 there - see [[kweebec-worldgen-v2-deep-pass]]).
- **The `KweebecNightmare_Grove` biome density is ALL-ADDITIVE + fail-safe**: `Sum(wideRamp, Multiplier(hillNoise,hillMask), Multiplier(caveMasks,-8), Multiplier(edgeMask,-100000))`. No top-level Min/Max, so any optional term degrading to `ConstantValueDensity(0.0)` just drops that feature - the disc NEVER airs out (the old `Min(ImportedCaves,...)` SPOF is gone). Hills come from a WIDE ramp curve (crossing y=80) + small (+-0.18) noise, NOT the sharp 1-block curve (which floods solid-to-sky when summed). `Cylinder` (`AxialCurve` Constant 1.0 -> Y-independent; `RadialCurve` over horizontal distance from origin) is the one radial primitive. Procedural caves only carve below ~Y64 OUTSIDE r70.
- **The biome also scatters ground cover** via nested `Props[]` (`Mesh2D` positions -> `Column` props floor-snapped by a `ColumnLinear` Scanner, widened to `MinY=-40/MaxY=60` so it still snaps onto the hilly ring): dead grass/tall grass, dead bush / twisted bramble / glowing purple mushroom. Atmosphere only, distinct from `ArenaBuilder`'s controlled tree ring. In the biome JSON `$NodeEditorMetadata`/`$Links` are dead node-editor metadata, not worldgen fields.
- **[`ArenaBuilder.build`](ArenaBuilder.java) stamps prefabs best-effort**: blocking prefab I/O on a background thread (`PrefabStore.findAssetPrefabPath` + `PrefabBufferUtil.getCached`), then each `PrefabUtil.paste` hops onto the world thread. A missing prefab is logged + skipped (the round still plays). [`Anchor`](Anchor.java) is the immutable `(x,y,z,yaw)` record + `horizontalDistanceSq`.
- **Pastes place the prefab ANCHOR at `FLOOR_Y` (`STAND_Y - 1`).** Buffer coords are stored anchor-relative (authored anchor at stored (0,0,0)) and `PrefabUtil.paste` puts stored (0,0,0) at the position. So the authored beats (anchor at their base) sit on the surface, while real vanilla trees (anchor a few root blocks below the trunk base) bury their roots and rise the trunk above - the intended look.
- **The decorative grove frame uses REAL vanilla dead-tree prefabs** (`FOLIAGE_PREFABS`: `Trees/Burnt_dead/Stage_2/...`, `Ash_Dead`, `Fir_Dead`, `Plants/Twisted_Wood/Fire`, `Trees/Dry_Dead`, `Trees/Oak/Stumps`), keys validated against hytale-resources. They paste on a deterministic ring OUTSIDE the shrine ring, clearing the gate/exit corridor + the hunter den so no clump lands on play space.
- **Underground shrines are a tunable feature** (`RuleSet.caveShrineCount`, default 2): `ChaseState` appends that many cave shrines (`ArenaLayout.caveShrineAnchors`, deterministic anchors inside the flat disc, distinct quadrants, capped at 4) AFTER the surface ring, so `allShrinesLit()` REQUIRES descending into every cave + returning. Each is carved by a `Relight_Shaft*` prefab pasted `force=true` (`paste(..., verbose, force)`: `force` routes through `PrefabUtil`'s `chunk.setBlock` - UNCONDITIONAL, writes `Empty` so the prefab CARVES the solid disc - instead of `placeBlock`). `ArenaBuilder` ALTERNATES the descent style per cave (by index) for variety: `[0]` `Relight_Shaft` (a facing-independent 3x3 spiral staircase), `[1]` `Relight_Shaft_Ladder` (the playtest-proven `Furniture_Kweebec_Ladder` descent). Each carries a baked shrine pillar (atomic, so an async re-carve can't erase it). Surface beats + foliage stay `force=false`.
- **The Heartwood Gate is DEFERRED**: it is NOT pasted at build; `ArenaBuilder.pasteGate(world)` is called from `ChaseMode.openGate` only when every shrine is lit (the dramatic reveal). The escape win is pure-anchor (`checkEscapes` crossing `GATE.z`), so the gate prefab is cosmetic.
- **The objective beats (surface shrine ring, exit, cave shaft) RE-PASTE at +4s and +9s into PREP** (`OBJECTIVE_REPASTE_DELAYS_SEC`) to beat a worldgen-overwrite race: on a busy relaunch the first paste lands before its chunks finish generating and gets clobbered (the later foliage already survives). Pastes are idempotent (fixed anchors + seeded RNG; the cave re-carve is a safe no-op even with a player inside), so a re-paste is harmless once a prefab has stuck.
