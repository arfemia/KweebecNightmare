# arena/ - the Chase grove skeleton (anchors + prefab stamping)

Router for `arena/`. The mod-root `CLAUDE.md` (loads with the kweebec subtree) has the Path-A-instance overview; the kweebec techspec (linked from the mod root) has the verified `PrefabUtil.paste` signature. Gameplay runs ENTIRELY off the anchors here, so a round is playable before any prefab pastes - the geometry is a visual layer, never a logic dependency.

- **[`ArenaLayout`](ArenaLayout.java) is the fixed coordinate authority** - `SPAWN`/`GATE`/`ESCAPE`/`HUNTER_DEN` + the cave anchors (`caveShrineAnchors(count, seed)` a seed-shuffled subset at `CAVE_STAND_Y`, used to CARVE the cave shafts) + (0.4.x) `SURFACE_WORLDGEN_SHRINES` (=3, the win-count term) and `WORLDGEN_SHRINE_XZ` (3 fixed Moonbloom-SUPPLY spots, NOT shrine positions - the shrine hosts SCATTER randomly, so these are just where the guaranteed Moonbloom clusters). **The surface shrine RING is GONE** - `shrineAnchors(...)` is vestigial. Surface shrines are WORLDGEN-SCATTERED host prefabs: a single `Mesh2D` `Type:Prefab` biome prop weighting the 3 `Shrine_*` hosts (each a copied vanilla crypt/ruin with the furnace baked as a 4-cell multiblock), discovered via the furnace interaction. **The trap (now in the `worldgen-v2` skill):** a worldgen prop path must resolve to a FOLDER (`Shrine_*/Shrine_*.prefab.json`, not a bare file - else it silently loads nothing). NOTE: a `List` fixed-position prop DOES work (the prop pass is world-positioned); the surface shrines use `Mesh2D` scatter by choice (random), but a `List` prop of fixed `{X,Y,Z}` (folder paths, Y in `[0,320]`) is the deterministic exact-count alternative if a fixed layout is wanted. `RuleSet.shrineCount` no longer drives placement. Proximity gates use `INTERACT_RADIUS_SQ` (rescue) and `ESCAPE_RADIUS_SQ`.
- **0.3.0 - corrupted village ruins + per-round variety.** The per-round world auto-seeds (the `instance.bson` `Seed` was removed; read back into `RoundInstance.worldSeed`), seeding the ring rotation, cave subset, AND the structure selection from ONE coherent seed. [`StructureCatalog`](StructureCatalog.java) (seeded selection) reads the asset-driven [`../asset/StructureConfig`](../asset/StructureConfig.java)`.all()` placement table; [`ExclusionMask`](ExclusionMask.java) (the shared gameplay keep-clear discs: spawn courtyard, shrine ring, gate corridor, den, cave anchors) keeps ruins off the beats. `ArenaBuilder.pasteStructures` (seed = `round.worldSeed()`) pastes 3-5 surface-snapped `Corrupted_*` ruins (cover/chokepoint/beacon/landmark), and the centerpiece is now `Corrupted_Well`. Ambient ruins also scatter gen-time via the biome `Props[1]` `Type:Prefab` prop (sparse, `LoadEntities:false`).
- **The arena is NATURAL ROLLING grove terrain (1C: the flat play disc is GONE), bounded by the finite EDGE CLIFF.** The Chase instance worldgen is the pack-authored `KweebecNightmare_Grove` (WorldStructure `Base`=80) whose biome density now rolls ~y72-88 across the WHOLE footprint, then a sheer cliff into the void at ~r112 (the design keeps the edge cliff). A SMALL r6 origin **spawn clearing** (a `Cylinder` gating the hill noise to 0 inside r6) keeps spawn flat + deterministic at y80 so `instance.bson` `SpawnProvider.SpawnPoint.Y`=80 stays valid - a courtyard, NOT the old disc (the original Y=65 buried players, the 0.2.0 bug). Off-center anchors are NOT flat-guaranteed, so their pastes FLOOR-SNAP to the real surface (see the ArenaBuilder bullet).
- **The `KweebecNightmare_Grove` biome density is ALL-ADDITIVE + fail-safe**: `Sum(wideRamp, hillNoise, Multiplier(caveMasks,-8), Multiplier(edgeMask,-100000))`. The two r70 flat-core `Cylinder` masks were REMOVED (1C) so hills + caves apply EVERYWHERE; only the r112 edge-cliff Cylinder remains. No top-level Min/Max, so any optional term degrading to `ConstantValueDensity(0.0)` just drops that feature - the ground NEVER airs out. Hills = a WIDE ramp curve (crossing y=80) summed with `Normalizer` noise (+-0.20, ~+-8 blocks), the noise wrapped in a `Multiplier` with the r6 spawn-clearing `Cylinder`; NOT the sharp 1-block curve (which floods solid-to-sky when summed). Caves carve only below ~Y66 (a fixed buffer under the lowest valley). See [[kweebec-worldgen-v2-deep-pass]] + the biome JSON `$Comment` hand-trace.
- **The biome scatters ALL ambient decoration via `Props[]`** (1C): ground cover (`Mesh2D` -> `Column` props floor-snapped by a `ColumnLinear` Scanner: dead grass/tall grass, dead bush / twisted bramble / glowing purple mushroom) AND the ambient blight **TREES** (a `Type:"Prefab"` prop over native `Trees/Poisoned`/`Ash_twisted`/`Petrified_Dead`/`Dry_Dead`/`Ash_Dead`, weighted + `SkipChance`-thinned, `BlockMask.DontReplace` so trunks don't gouge soil, mirroring the proven vanilla `Example_Prop` shape). The old runtime `ArenaBuilder` foliage ring is GONE (the design's "trees from worldgen, not manual paste"; it also cost a world-thread paste stall + the worldgen race). In the biome JSON `$NodeEditorMetadata`/`$Links` are dead node-editor metadata. The biome JSON is NOT gradle-compiled - validate terrain/scatter in-game. **FULL ATMOSPHERIC PASS** (Seam 1 of [[kweebec-worldgen-both-seams]]): the grove biome now has SIX `Props[]` layers - `KN_Trees` (widened dead/ash/petrified/burnt palette + rare Ash_twisted Large/Giant landmarks), `KN_Ruins` (pack `Corrupted_*`), `KN_Grass`, `KN_Brush` (bush/bramble/mushroom), plus the new `KN_Debris` (fallen logs / stumps / petrified logs / mossy boulders, Scale 14) and `KN_Graves` (Cursed_Crypt tombstones + sign-post, Scale 40, rare). All new layers are CLONED from the proven `Corrupted_*` Prefab-prop shape (a malformed Prefab prop airs out the WHOLE biome) and every prefab path was verified to resolve. The big `Crypt_Entrance` set-piece is deliberately NOT worldgen-scattered (it carves a deep pit; a random drop onto a shrine/spawn would break the chase) - it belongs on the runtime, `ExclusionMask`-protected seam.
- **[`ArenaBuilder.build`](ArenaBuilder.java) stamps the RUNTIME beats**: the exit, the CAVE shafts (force-carved), the centerpiece + corrupted ruins; the gate is deferred. **Surface shrines are NO LONGER stamped here** - they are WORLDGEN-SCATTERED host prefabs (a `Mesh2D` `Type:Prefab` biome prop over the `Shrine_*` folders), so `placeShrineBlock`/`setShrineFurnace` are GONE (a single `setBlock` of the 2x2 furnace rendered wrong; the furnace is baked into the prefab instead). Best-effort: blocking prefab I/O on a background thread, then each paste hops to the world thread (a missing prefab is logged + skipped). **Each paste FLOOR-SNAPS to the real surface**: `paste(...)` probes the local top-solid Y with `ziggfreed-common`'s `SurfaceProbe` inside `world.execute` (degrading to `at.y-1` on a miss). **The probe SKIPS surface decoration** (`SURFACE_DECORATION_LISTS` = `TreeWoodAndLeaves` + `AllScatter` via `BlockTypeLists.keys(...)`) so a beat lands on ground, not a tree/leaf. **CORRECTION to the old both-seams claim**: objectives CAN ride worldgen when expressed as a registered interactable BLOCK (the furnace) baked into a prefab + discovered via its interaction (no position recompute / placement registry needed) - see [[kweebec-worldgen-both-seams]] + the `hytale-modding` `worldgen-v2.md` objective-as-block recipe (reusable for the MMO). [`Anchor`](Anchor.java) is the `(x,y,z,yaw)` record + `horizontalDistanceSq`.
- **Underground shrines are a tunable feature** (`RuleSet.caveShrineCount`, default 2): `ArenaBuilder.pasteObjectives` carves that many cave shafts directly from `ArenaLayout.caveShrineAnchors(count, seed)` (deterministic anchors SPREAD to r~38 across the four quadrants, capped at 4; `ExclusionMask` auto-tracks them) - the cave count is part of `ChaseState.totalShrines`, so `allShrinesLit()` REQUIRES descending into every cave + returning. `pasteCaveShaft` FLOOR-SNAPS each shaft top to the LOCAL surface (`SurfaceProbe`) and force-carves it (`force=true` -> `PrefabUtil` `chunk.setBlock`, UNCONDITIONAL, writes `Empty` so the prefab CARVES the rock). **The shrine FURNACE is BAKED into the shaft prefab at the chamber** (the cyan beam + the runtime `setShrineFurnace` are GONE), discovered via its interaction like the surface shrines. Idempotent re-carve: the resolved surface Y is remembered per-cave in `ChaseState.caveCarveY` so a +4s/+9s re-paste reuses it instead of re-probing (which would stack a second shaft). Descent style alternates per cave: `[0]` `Relight_Shaft` (a facing-independent spiral staircase), `[1]` `Relight_Shaft_Ladder`. Surface (worldgen) beats are `force=false`.
- **The Heartwood Gate is DEFERRED**: it is NOT pasted at build; `ArenaBuilder.pasteGate(world)` is called from `ChaseMode.openGate` only when every shrine is lit (the dramatic reveal). The escape win is pure-anchor (`checkEscapes` crossing `GATE.z`), so the gate prefab is cosmetic.
- **The runtime beats (exit + cave shafts) RE-PASTE at +4s and +9s into PREP** (`OBJECTIVE_REPASTE_DELAYS_SEC`) to beat a worldgen-overwrite race: on a busy relaunch the first paste can land before its chunks finish generating and get clobbered. The cave re-carve is idempotent via `ChaseState.caveCarveY` (the stored surface Y -> the same carve, never a stacked shaft). **Surface shrines are worldgen-placed, so they are NOT in the re-paste loop** (worldgen does not race itself).
