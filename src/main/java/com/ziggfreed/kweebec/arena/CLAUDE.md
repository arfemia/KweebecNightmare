# arena/ - the Chase grove skeleton (anchors + prefab stamping)

Router for `arena/`. The mod-root `CLAUDE.md` (loads with the kweebec subtree) has the Path-A-instance overview; the kweebec techspec (linked from the mod root) has the verified `PrefabUtil.paste` signature. Gameplay runs ENTIRELY off the anchors here, so a round is playable before any prefab pastes - the geometry is a visual layer, never a logic dependency.

- **[`ArenaLayout`](ArenaLayout.java) is the fixed coordinate authority** - `SPAWN`/`GATE`/`ESCAPE`/`HUNTER_DEN` plus the shrine ring (`shrineAnchors(count)`, deterministic so every client agrees on the layout). All anchors sit at `STAND_Y`; `count` comes from `RuleSet.shrineCount(partySize)`. Proximity gates use `INTERACT_RADIUS_SQ` (shrine/rescue) and `ESCAPE_RADIUS_SQ`.
- **The floor Y MUST match the worldgen, or players spawn inside terrain.** The Chase instance uses vanilla `Default_Flat` (WorldStructure `Base` = 80), so the walkable surface top is world Y=80 (`STAND_Y`) over a top solid block at Y=79 (`FLOOR_Y`); `instance.bson` `SpawnProvider.SpawnPoint.Y` MUST agree (80). The original Y=65 buried players ~15 blocks underground (the 0.2.0 bug). The `Default_Flat` movement gym confirms spawn Y=80. Re-derive `FLOOR_Y`/`STAND_Y` from `Base` if you ever swap to a pack-authored `HytaleGenerator`.
- **[`ArenaBuilder.build`](ArenaBuilder.java) stamps prefabs best-effort**: blocking prefab I/O on a background thread (`PrefabStore.findAssetPrefabPath` + `PrefabBufferUtil.getCached`), then each `PrefabUtil.paste` hops onto the world thread. Pastes at `STAND_Y - 1` (= `FLOOR_Y`). A missing prefab is logged at FINE + skipped (the round still plays). [`Anchor`](Anchor.java) is the immutable `(x,y,z,yaw)` record + `horizontalDistanceSq`.
