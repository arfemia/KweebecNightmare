# arena/ - the Chase grove skeleton (anchors + prefab stamping)

Router for `arena/`. The mod-root `CLAUDE.md` (loads with the kweebec subtree) has the Path-A-instance overview; the kweebec techspec (linked from the mod root) has the verified `PrefabUtil.paste` signature. Gameplay runs ENTIRELY off the anchors here, so a round is playable before any prefab pastes - the geometry is a visual layer, never a logic dependency.

- **[`ArenaLayout`](ArenaLayout.java) is the fixed coordinate authority** - `SPAWN`/`GATE`/`ESCAPE`/`HUNTER_DEN` plus the shrine ring (`shrineAnchors(count)`, deterministic so every client agrees on the layout) AND `SHRINE_CAVE` (the one sub-surface objective). Surface anchors sit at `STAND_Y`; `SHRINE_CAVE` sits at `CAVE_STAND_Y` (the chamber stand level); `count` comes from `RuleSet.shrineCount(partySize)`. Proximity gates use `INTERACT_RADIUS_SQ` (shrine/rescue) and `ESCAPE_RADIUS_SQ`.
- **The arena is NATURAL ROLLING grove terrain (1C: the flat play disc is GONE), bounded by the finite EDGE CLIFF.** The Chase instance worldgen is the pack-authored `KweebecNightmare_Grove` (WorldStructure `Base`=80) whose biome density now rolls ~y72-88 across the WHOLE footprint, then a sheer cliff into the void at ~r112 (the design keeps the edge cliff). A SMALL r6 origin **spawn clearing** (a `Cylinder` gating the hill noise to 0 inside r6) keeps spawn flat + deterministic at y80 so `instance.bson` `SpawnProvider.SpawnPoint.Y`=80 stays valid - a courtyard, NOT the old disc (the original Y=65 buried players, the 0.2.0 bug). Off-center anchors are NOT flat-guaranteed, so their pastes FLOOR-SNAP to the real surface (see the ArenaBuilder bullet).
- **The `KweebecNightmare_Grove` biome density is ALL-ADDITIVE + fail-safe**: `Sum(wideRamp, hillNoise, Multiplier(caveMasks,-8), Multiplier(edgeMask,-100000))`. The two r70 flat-core `Cylinder` masks were REMOVED (1C) so hills + caves apply EVERYWHERE; only the r112 edge-cliff Cylinder remains. No top-level Min/Max, so any optional term degrading to `ConstantValueDensity(0.0)` just drops that feature - the ground NEVER airs out. Hills = a WIDE ramp curve (crossing y=80) summed with `Normalizer` noise (+-0.20, ~+-8 blocks), the noise wrapped in a `Multiplier` with the r6 spawn-clearing `Cylinder`; NOT the sharp 1-block curve (which floods solid-to-sky when summed). Caves carve only below ~Y66 (a fixed buffer under the lowest valley). See [[kweebec-worldgen-v2-deep-pass]] + the biome JSON `$Comment` hand-trace.
- **The biome scatters ALL ambient decoration via `Props[]`** (1C): ground cover (`Mesh2D` -> `Column` props floor-snapped by a `ColumnLinear` Scanner: dead grass/tall grass, dead bush / twisted bramble / glowing purple mushroom) AND the ambient blight **TREES** (a `Type:"Prefab"` prop over native `Trees/Poisoned`/`Ash_twisted`/`Petrified_Dead`/`Dry_Dead`/`Ash_Dead`, weighted + `SkipChance`-thinned, `BlockMask.DontReplace` so trunks don't gouge soil, mirroring the proven vanilla `Example_Prop` shape). The old runtime `ArenaBuilder` foliage ring is GONE (the design's "trees from worldgen, not manual paste"; it also cost a world-thread paste stall + the worldgen race). In the biome JSON `$NodeEditorMetadata`/`$Links` are dead node-editor metadata. The biome JSON is NOT gradle-compiled - validate terrain/scatter in-game.
- **[`ArenaBuilder.build`](ArenaBuilder.java) stamps ONLY the gameplay beats** (shrine ring, exit, cave shafts; the gate is deferred), best-effort: blocking prefab I/O on a background thread, then each paste hops to the world thread (a missing prefab is logged + skipped, round still plays). **Each paste FLOOR-SNAPS to the real surface**: `paste(...)` probes the local top-solid Y with `ziggfreed-common`'s `SurfaceProbe` (`com.ziggfreed.common.world.SurfaceProbe`) inside `world.execute`, landing the prefab there (degrading to `at.y-1` on a probe miss) - a hardcoded Y would float/bury every beat on rolling terrain. [`Anchor`](Anchor.java) is the `(x,y,z,yaw)` record + `horizontalDistanceSq`; [`ShrineState`](../mode/chase/ShrineState.java)'s anchor is now RE-POINTABLE (`setAnchor`) so a cave's chamber Y can be snapped.
- **Underground shrines are a tunable feature** (`RuleSet.caveShrineCount`, default 2): `ChaseState` appends that many cave shrines (`ArenaLayout.caveShrineAnchors`, deterministic anchors in distinct quadrants, capped at 4) AFTER the surface ring, so `allShrinesLit()` REQUIRES descending into every cave + returning. `ArenaBuilder.pasteCaveShaft` FLOOR-SNAPS each shaft top to the LOCAL surface (`SurfaceProbe`), force-carves it (`force=true` -> `PrefabUtil` `chunk.setBlock`, UNCONDITIONAL, writes `Empty` so the prefab CARVES the rock), and RE-POINTS the cave `ShrineState`'s chamber stand Y to `surface - CAVE_SHAFT_DEPTH` (so `ChaseMode`'s underground channel Y-band matches the carved chamber on rolling terrain; falls back to the old Y66 on a probe miss). Descent style alternates per cave (by index): `[0]` `Relight_Shaft` (a facing-independent 3x3 spiral staircase), `[1]` `Relight_Shaft_Ladder` (the playtest-proven `Furniture_Kweebec_Ladder` descent). Each carries a baked shrine pillar (atomic, so an async re-carve can't erase it). Surface beats stay `force=false`.
- **The Heartwood Gate is DEFERRED**: it is NOT pasted at build; `ArenaBuilder.pasteGate(world)` is called from `ChaseMode.openGate` only when every shrine is lit (the dramatic reveal). The escape win is pure-anchor (`checkEscapes` crossing `GATE.z`), so the gate prefab is cosmetic.
- **The objective beats (surface shrine ring, exit, cave shaft) RE-PASTE at +4s and +9s into PREP** (`OBJECTIVE_REPASTE_DELAYS_SEC`) to beat a worldgen-overwrite race: on a busy relaunch the first paste can land before its chunks finish generating and get clobbered. Pastes are idempotent (fixed anchors + a fixed surface probe -> the same Y; the cave re-carve + chamber re-point are safe no-ops even with a player inside), so a re-paste is harmless once a prefab has stuck.
