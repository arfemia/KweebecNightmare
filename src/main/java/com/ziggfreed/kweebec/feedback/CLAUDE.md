# feedback/ - round HUD, heartbeat, titles, toasts (cosmetic only)

Router for `feedback/`. Everything here is cosmetic and world-thread; every channel is try-guarded so a missing asset can never throw into the round loop. The floating combat-text / 3D-sound packet mechanics live in the parent repo's `hytale-modding` skill.

- **[`RoundFeedback`](RoundFeedback.java) is the single fan-out point** for titles, toasts (Danger/Success/Warning/Default), and custom-HUD lifecycle: `installHud` adds the HUD and strips the native HUD; `restoreHud` resets it on exit/round-end. Every string is keyed through [`../i18n/Lang`](../i18n/Lang.java). **It now DELEGATES the engine plumbing to `ziggfreed-common`** - toasts to `feedback.Notify` (danger/success/warning/def), banners to `feedback.EventTitles.show`, and the HUD install/strip/restore to `ui.CustomHudHelper` (it passes `CustomHudHelper.defaultKeptHud()`, which is byte-identical to the old local `KEPT_HUD`). It keeps ONLY the mod-specific policy: the `Lang.msg(...)` keying and the generic `installCustomHud(Function factory)` form the PvP modes reuse (common has no factory equivalent). Common's helpers already try-guard every channel, so the redundant local try-guards were dropped.
- **[`NightmareHud`](NightmareHud.java) is the custom overlay** (`CustomUIHud`, `HUD_KEY = "kweebecnightmare.hud"`); `pushState(...)` sends partial `cb.set("#Elem.Text", ...)` updates. **The element-id contract (`#Timer`/`#Objective`/`#Shrines`/`#Alive`/`#Corruption`/`#Extraction`) is shared with the paired `Common/UI/Custom/Hud/KweebecNightmareHud.ui`** - keep the ids in sync. `#Extraction` is the co-op extraction line: filled only during the gate-open ESCAPE climax (the "X/Y on the platform" count + the hold countdown via `HUD_EXTRACTION_HOLD`/`HUD_EXTRACTION_WAIT`), blank otherwise (its `.ui` default is empty, so no per-tick clear is needed). If the HUD shows RAW KEYS on screen, that is the `.lang` prefix bug in [`../i18n/CLAUDE.md`](../i18n/CLAUDE.md), NOT a HUD fault.
- **[`BossHud`](BossHud.java) is the Warden capstone health bar** (`CustomUIHud`, `HUD_KEY = "kweebecnightmare.boss.hud"`), installed on every survivor by `boss/BossController` when the Warden spawns, stripped on its death / round end; `pushHealth(...)` rewrites `#BossName`/`#BossPhase` `.Text` + the HP bar each tick, paired with `Common/UI/Custom/Hud/KweebecBossHud.ui`. **CustomUI `cb.set` SELECTOR RULE (a shipped client-crash bug, fixed twice):** a `Set` selector must target a RUNTIME-SETTABLE markup property, else the client HARD-DISCONNECTS with "Failed to apply CustomUI HUD commands (selector doesn't match a markup property)". **A Group's `.Width` is NOT runtime-settable** - it is a layout INPUT, and `cb.set("#El.Width", px)` crashes the client regardless of the element's anchor shape (empirically crash-proven on the boss bar - the earlier "`.Width` works, `.Anchor.Width` does not" note was WRONG; the cited MMO `ClassHubPage #AdvancementProgressBar.Width` is latently broken too, just never exercised). **Proven settable selectors:** `.Text` (Labels), `.Background`/`.Background.Color` + `.Visible` (any element; `.Visible` used heavily in MMO pages), and `.Value` (0..1) on a native `ProgressBar` widget (the engine idiom - MMO `ViewXpPage #CardProgress.Value`, native `MemoriesPage #MemoriesProgressBar.Value`). A proportional bar therefore has two options: a native `ProgressBar` + `.Value` (but it needs a `BarTexturePath` we'd red-X on), OR - what BossHud does - a row of N fixed colour-Group SEGMENTS toggled via `.Visible` (texture-free, cannot red-X; the HP bar is 20 `#BossHpSegN` segments, `ceil(fraction*20)` shown).
- **[`HeartbeatService`](HeartbeatService.java) is a server-pulsed 3D one-shot** played to each active survivor at their OWN position (`SoundUtil.playSoundEvent3d` with a self-only predicate), re-armed on a tightening interval driven by `ChaseState.corruptionTier()` (there is no engine StopSound for a loop - the dread bed is repeated one-shots; stopping = stop scheduling). Resolves the pack `KweebecNightmare_Heartbeat` then vanilla fallbacks and caches ONLY a real index.
- **[`ScareDirector`](ScareDirector.java) is the horror conductor (0.3.0)**, driven 1 Hz from `ChaseMode.tick` (after the hunter tick, non-PREP). Per ACTIVE survivor it computes the min distance to any hunter (`round.hunterController().hunterPositions(store)`) + `ChaseState.corruptionTier()` -> a proximity band, looks up the band via ziggfreed-common's `BandedEffectConfig.bandFor(band,tier)` / `.oneShot()` (the migrated-to-common `BandedEffect` asset, content at `Server/ZiggfreedCommon/BandedEffects/`) and applies its EntityEffect (`KweebecNightmare_Dread_1/2/3`) to the survivor via `EffectControllerComponent.addEffect/removeEffect` WITH HYSTERESIS (mirrors `AiHunterController.applySpeed`, but on players). **Note (`applyBand` adoption DEFERRED, intentional):** the `applyVignette` remove/add/track dance looks like ziggfreed-common's `EntityEffectService.applyBand`, but it is NOT migrated to it. `applyVignette` selects the effect via the TIER-AWARE `BandedEffectConfig.bandFor(targetBand, tier)` (highest authored band `<= targetBand` whose `minTier <= tier`, indexed by band VALUE 1..3 with band 0 = clear), whereas `applyBand` consumes a tier-AGNOSTIC `EffectBandLadder` (from `toLadder()`) indexed by compacted RUNG position - so passing `targetBand` straight in would shift every vignette one band heavier (band 1 -> Dread_2, band 3 -> none) and drop the `minTier`/sparse-table semantics. Reproducing the exact selection would mean rebuilding a ladder per-tick or losing the tier dimension (the "contorting the tier logic" the lift paradigm warns against), so the hand-rolled swap stays to preserve the corruption-tier vignette pacing. On entering the closest band (or via `onAlert`) it fires a throttled, PER-PRESET jumpscare resolved from `round.ruleSet()` jumpscare knobs (`jumpscareEnabled`/`jumpscareBeatId`/`jumpscareShakeIntensity`/`jumpscareCooldownSeconds`, authored per game mode in `RoundPresetAsset` + the `Presets/*.json` live layer): the scare BEAT (a ziggfreed-common `BandedEffect` one-shot, `.byId()` else `.oneShot()`) bundles the overlay `EntityEffect` (`KweebecNightmare_Jumpscare`, a real NATIVE full-screen `ScreenEffect` PNG at `Common/ScreenEffects/KweebecNightmare_Jumpscare.png`) + the `SFX_Hedera_Scream` stinger + the camera shake (`BandedEffectAsset.shakeId()`/`shakeIntensity()` -> `CameraShakeService`; the `KweebecNightmare_JumpscareShake` `CameraEffect`), plus a `RoundFeedback` title. Native screen-effect + camera-shake mechanism: [[hytale-screen-effects-and-camera-shake]]. A separate daemon-scheduler whisper layer (`SFX_Emit_Forgotten_Whispers`, frequency rising with tier) mirrors `HeartbeatService`. All validate-before-use + try-guarded. **The parent wires `ScareDirector.clear(round, store)` into `RoundService.teardown`** (removes vignettes + per-player state + the whisper schedule so no dread effect rides out on an ejected player) and `ScareDirector.shutdown()` at round-service shutdown. The tier/band -> effect mapping is asset-driven (no hardcoded table).
