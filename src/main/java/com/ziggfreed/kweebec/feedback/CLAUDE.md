# feedback/ - round HUD, heartbeat, titles, toasts (cosmetic only)

Router for `feedback/`. Everything here is cosmetic and world-thread; every channel is try-guarded so a missing asset can never throw into the round loop. The floating combat-text / 3D-sound packet mechanics live in the parent repo's `hytale-modding` skill.

- **[`RoundFeedback`](RoundFeedback.java) is the single fan-out point** for titles (`EventTitleUtil`), toasts (`NotificationUtil`, Danger/Success/Warning), and custom-HUD lifecycle: `installHud` adds the HUD and strips the native HUD down to `KEPT_HUD`; `restoreHud` resets it on exit/round-end. Every string is keyed through [`../i18n/Lang`](../i18n/Lang.java).
- **[`NightmareHud`](NightmareHud.java) is the custom overlay** (`CustomUIHud`, `HUD_KEY = "kweebecnightmare.hud"`); `pushState(...)` sends partial `cb.set("#Elem.Text", ...)` updates. **The element-id contract (`#Timer`/`#Objective`/`#Shrines`/`#Alive`/`#Corruption`/`#Extraction`) is shared with the paired `Common/UI/Custom/Hud/KweebecNightmareHud.ui`** - keep the ids in sync. `#Extraction` is the co-op extraction line: filled only during the gate-open ESCAPE climax (the "X/Y on the platform" count + the hold countdown via `HUD_EXTRACTION_HOLD`/`HUD_EXTRACTION_WAIT`), blank otherwise (its `.ui` default is empty, so no per-tick clear is needed). If the HUD shows RAW KEYS on screen, that is the `.lang` prefix bug in [`../i18n/CLAUDE.md`](../i18n/CLAUDE.md), NOT a HUD fault.
- **[`HeartbeatService`](HeartbeatService.java) is a server-pulsed 3D one-shot** played to each active survivor at their OWN position (`SoundUtil.playSoundEvent3d` with a self-only predicate), re-armed on a tightening interval driven by `ChaseState.corruptionTier()` (there is no engine StopSound for a loop - the dread bed is repeated one-shots; stopping = stop scheduling). Resolves the pack `KweebecNightmare_Heartbeat` then vanilla fallbacks and caches ONLY a real index.
- **[`ScareDirector`](ScareDirector.java) is the horror conductor (0.3.0)**, driven 1 Hz from `ChaseMode.tick` (after the hunter tick, non-PREP). Per ACTIVE survivor it computes the min distance to any hunter (`round.hunterController().hunterPositions(store)`) + `ChaseState.corruptionTier()` -> a proximity band, looks up the band via ziggfreed-common's `BandedEffectConfig.bandFor(band,tier)` / `.oneShot()` (the migrated-to-common `BandedEffect` asset, content at `Server/ZiggfreedCommon/BandedEffects/`) and applies its EntityEffect (`KweebecNightmare_Dread_1/2/3`) to the survivor via `EffectControllerComponent.addEffect/removeEffect` WITH HYSTERESIS (mirrors `AiHunterController.applySpeed`, but on players). On entering the closest band (or via `onAlert`) it fires a throttled, PER-PRESET jumpscare resolved from `round.ruleSet()` jumpscare knobs (`jumpscareEnabled`/`jumpscareBeatId`/`jumpscareShakeIntensity`/`jumpscareCooldownSeconds`, authored per game mode in `RoundPresetAsset` + the `Presets/*.json` live layer): the scare BEAT (a ziggfreed-common `BandedEffect` one-shot, `.byId()` else `.oneShot()`) bundles the overlay `EntityEffect` (`KweebecNightmare_Jumpscare`, a real NATIVE full-screen `ScreenEffect` PNG at `Common/ScreenEffects/KweebecNightmare_Jumpscare.png`) + the `SFX_Hedera_Scream` stinger + the camera shake (`BandedEffectAsset.shakeId()`/`shakeIntensity()` -> `CameraShakeService`; the `KweebecNightmare_JumpscareShake` `CameraEffect`), plus a `RoundFeedback` title. Native screen-effect + camera-shake mechanism: [[hytale-screen-effects-and-camera-shake]]. A separate daemon-scheduler whisper layer (`SFX_Emit_Forgotten_Whispers`, frequency rising with tier) mirrors `HeartbeatService`. All validate-before-use + try-guarded. **The parent wires `ScareDirector.clear(round, store)` into `RoundService.teardown`** (removes vignettes + per-player state + the whisper schedule so no dread effect rides out on an ejected player) and `ScareDirector.shutdown()` at round-service shutdown. The tier/band -> effect mapping is asset-driven (no hardcoded table).
