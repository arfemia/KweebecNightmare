# round/ - round lifecycle, state machine, RuleSet, cocoon-shared state

Router for `round/`. The mod-root `CLAUDE.md` Architecture section + the parent repo's `hytale-api` skill (Store/Ref, `world.execute`, instance lifecycle) are the deeper authority; the kweebec techspec has the verified call sequences. This package OWNS the round engine because the engine has no player-party API.

- **[`RoundService`](RoundService.java) is the single mutating authority** (singleton): `startChase` (spawn instance + teleport the party in), `resolve` (idempotent win/lose teardown - `RoundInstance.claimResolution` guards double-resolve), `exit`, `endAll`. Public methods are safe from any thread; all world-bound work hops via `world.execute`. Each party member captures their OWN return transform so non-initiators return where THEY were, not the initiator's spot.
- **[`RoundStateMachine`](RoundStateMachine.java) drives 1 Hz OFF-thread, then hops on.** A daemon scheduler polls every live `ACTIVE` round and dispatches `ChaseMode.tick` via `world.execute`; a returned `Outcome` triggers `RoundService.resolve`. NEVER read a Store/Ref off the scheduler thread.
- **[`InstanceLifecycle`](InstanceLifecycle.java) is the Path-A seam.** `spawnInstance` is off-thread-safe (consume via `whenComplete`, NEVER `.join()` on a world thread); `teleportIn`/`exit` run on the player's CURRENT world thread; `safeRemove` flags removal-when-empty; `removeWorldOffThread` uses a dedicated daemon because `removeWorld` self-joins and would deadlock on the target world's own thread. [`CleanupTicker`](CleanupTicker.java) is the eviction backstop when authored `RemovalConditions` have not fired.
- **[`RoundInstance`](RoundInstance.java) is the mutable per-round state.** Lifecycle fields are `volatile` (the off-thread scheduler reads `state`/`world` before hopping); per-player gameplay state is mutated only on the world thread. HUD handles are stored as `Object` to keep the class engine-light. [`PlayerRoundState`](PlayerRoundState.java) carries cocoon/escape/left/downs/bleed-out/rescue + the entry `gameModeApplied` flag; `isActive()` = present AND not cocooned/escaped/left.
- **[`RoundInventoryGuard`](RoundInventoryGuard.java) is the inventory preserve/restore seam** - the consumer wiring over ziggfreed-common's `InventorySnapshotStore`. On entry (`onEnter`, called from `RoundService.startChase` on each player's CURRENT world thread before teleport) it snapshots + persists + strips the survivor's WHOLE inventory (`InventoryStripPolicy.STRIP_ALL`) unless the mode is `InventoryMode.KEEP`; on exit (`restore`, from `scheduleOverworldResync` once the player is back in the overworld) it restores the exact entry state and drops any in-round loot. Crash/disconnect net: `onPlayerReady` re-applies a leftover snapshot on next login for anyone who entered but was never restored in-instance (it skips players still bound to a live round). The persisted file lives under the plugin data dir (`init` in plugin setup). The common primitive supports partial keeps (e.g. armor) + item whitelists/blacklists; kweebec strips all.
- **[`RuleSet`](RuleSet.java) is the replayability pillar** - immutable, built once per round, but now **SOURCED from the asset layer**: `RoundService.startChase` resolves it via [`../integration/KweebecNightmareAPI`](../integration/KweebecNightmareAPI.java)`.resolveRuleSet(presetId)`, which composes `defaults < pack < owner < runtime` ([`../asset/PresetConfig`](../asset/CLAUDE.md) folds the pack-authored [`RoundPresetAsset`](../asset/RoundPresetAsset.java) over the [`DefaultPresets`](../asset/DefaultPresets.java) jar baselines; the MMO scales/overrides at runtime via the API). [`RoundPreset`](RoundPreset.java) is now a thin compatibility SHIM delegating to `PresetConfig`, NOT the schema authority. Every stake (rescue style, downs, bleed-out, hunter count/speed, shrine count, cap, corruption ramp, + `InventoryMode`/`HunterArchetype`) is a knob a pack OR an installed MMO can author/scale. (The cross-cutting reward-on-exit policy moved to the common `InstancePreset`; `RuleSet` no longer carries `RewardOnExit`.) [`KweebecMode`](KweebecMode.java) maps a mode -> `instanceName` + lang keys (only `CHASE` is built; `SURVIVAL` is reserved).
