# atmosphere/ - dark-midnight weather + dread music bed

Router for `atmosphere/`. World-thread-only (self-hops via `world.execute`), fully try-guarded.

- **[`AtmosphereService.lock`](AtmosphereService.java)** freezes the world into a dark midnight: it keeps only the mod-specific POLICY here (the `DARK_WEATHER_CANDIDATES` list + `firstValidWeather()` choice - pack weather first, then vanilla Void/Blood-Moon/Terror) and DELEGATES the time + weather ENGINE mechanism to ziggfreed-common's [`world.AtmosphereService.lock(world, 0.0, chosenWeatherId)`](../../../../../../../../ziggfreed-common/src/main/java/com/ziggfreed/common/world/AtmosphereService.java) (set+pause the clock at midnight via `WorldTimeResource.setDayTime` + `WorldConfig.setGameTimePaused(true)` + `markChanged`, then validate + force the chosen weather; a `null` choice leaves the weather untouched). Common self-hops to the world thread, so `lock` is safe to call from any thread.
- **Validate every asset id before applying.** A `getIndex == Integer.MIN_VALUE` means the asset is missing - skip it, never set a bad id (an unknown weather blanks the sky). The same validate-then-use rule governs sounds in [`../feedback/`](../feedback/CLAUDE.md) and the cocoon effect in [`../death/`](../death/CLAUDE.md).
- **[`MusicBedService.applyFor`](MusicBedService.java)** forces the dread bed and WORKS in playtest. It keeps only the mod-specific POLICY here (the `DREAD_MUSIC_CANDIDATES` + per-tier `TIER_MUSIC_CANDIDATES` ladders, the tier-to-container selection, and the per-player one-shot apply / retry logic) and DELEGATES the forced-music ENGINE mechanism to ziggfreed-common's [`world.ForcedMusicService`](../../../../../../../../ziggfreed-common/src/main/java/com/ziggfreed/common/world/ForcedMusicService.java) (`applyFor(store, ref, containerId)` resolves the `MusicContainer` index, sets the `ForcedMusicTracker`, and pushes the `UpdateForcedMusic` packet directly; `clearFor(store, ref)` sends index 0 to restore the default bed). The kweebec layer resolves a candidate id (caching only a registered id, never latching), then hands it to common per player; `MC_Zone4_Dark` is the resolved dread container. `applyFor` returns false to retry next tick when no container is registered yet; `clear(round)` clears the override for every player on exit.
- **Ambience lever = the biome Environment.** The grove biome [`KweebecNightmare_Grove.json`](../../../../../../resources/Server/HytaleGenerator/Biomes/KweebecNightmare_Grove/KweebecNightmare_Grove.json) sets `EnvironmentProvider` -> `Env_Void`, so its `AmbFX_Void` void-static loop plays (and `SpawnDensity 0`). Ambience is matched by `AmbienceFX.Conditions.EnvironmentIds`, so to change or silence the ambience, swap the biome `Environment`, not any Java. Both dread music + `Env_Void` ambience verified this session, alongside the relentless no-loop hunter chase, the death-eject fix, and the gate/exit win.
