# atmosphere/ - dark-midnight weather + dread music bed

Router for `atmosphere/`. World-thread-only (self-hops via `world.execute`), fully try-guarded.

- **[`AtmosphereService.lock`](AtmosphereService.java)** freezes the world into a dark midnight: `WorldTimeResource.setDayTime` to midnight (`dayTime` is a day-fraction from hour 0, where 0.0 = midnight / darkest) + `WorldConfig.setGameTimePaused(true)` so night never advances, then forces the first dark weather that VALIDATES (`Weather.getAssetMap().getIndex != Integer.MIN_VALUE`) - the pack weather first, then vanilla Void/Blood-Moon/Terror fallbacks. `markChanged()` after.
- **Validate every asset id before applying.** A `getIndex == Integer.MIN_VALUE` means the asset is missing - skip it, never set a bad id (an unknown weather blanks the sky). The same validate-then-use rule governs sounds in [`../feedback/`](../feedback/CLAUDE.md) and the cocoon effect in [`../death/`](../death/CLAUDE.md).
- **[`MusicBedService.applyFor`](MusicBedService.java)** forces the dread bed and WORKS in playtest. The lever is the `UpdateForcedMusic` packet (id 151, a single int container index): set the player's `ForcedMusicTracker` `currentContainerIndex` AND push `UpdateForcedMusic` DIRECTLY through `PlayerRef.getPacketHandler().write` (mechanism-independent - `ForcedMusicSystems.Tick` is global so it DOES run in instance worlds, but the direct send does not depend on it; on a successful send we also set `lastSentContainerIndex` so Tick will not duplicate). `MC_Zone4_Dark` is the resolved dread container (`DREAD_MUSIC_CANDIDATES`, validate before use). The earlier silence had two causes, both fixed: an index-0 latch (`applyFor`/`resolveMusic` now return false / re-resolve and NEVER cache 0, so the caller retries next tick) and a client-side music-off setting. `clear(round)` sends index 0 to every player on exit.
- **Ambience lever = the biome Environment.** The grove biome [`KweebecNightmare_Grove.json`](../../../../../../resources/Server/HytaleGenerator/Biomes/KweebecNightmare_Grove/KweebecNightmare_Grove.json) sets `EnvironmentProvider` -> `Env_Void`, so its `AmbFX_Void` void-static loop plays (and `SpawnDensity 0`). Ambience is matched by `AmbienceFX.Conditions.EnvironmentIds`, so to change or silence the ambience, swap the biome `Environment`, not any Java. Both dread music + `Env_Void` ambience verified this session, alongside the relentless no-loop hunter chase, the death-eject fix, and the gate/exit win.
