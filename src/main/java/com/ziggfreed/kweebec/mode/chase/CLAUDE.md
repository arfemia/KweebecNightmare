# mode/chase/ - the Chase round loop (Relight & Escape)

Router for `mode/chase/`. Driven 1 Hz from [`../../round/RoundStateMachine`](../../round/RoundStateMachine.java) on the instance world thread; stateless (all state lives on [`../../round/RoundInstance`](../../round/RoundInstance.java) + [`ChaseState`](ChaseState.java)).

- **[`ChaseMode.tick`](ChaseMode.java) is the whole loop**: PREP (a breath) -> RITUAL (spawn the hunter + heartbeat, survivors channel shrines, corruption climbs with time + per shrine) -> ESCAPE (the last shrine opens the gate, the hunter hard-locks onto the nearest survivor, crossing the gate or reaching the exit wins). `lazyPlayerSetup` handles async arrival/disconnect via player ABSENCE (two grace windows: longer for still-teleporting, short for entered-then-dropped) + first-arrival HUD install + the dread music bed one-shot.
- **The shrine ritual is a deliberate RISK** (`handleShrines`): the channeller becomes `ChaseState.loudestChanneller()`, the hunter's PRIORITY target (the noise draws it, see `AiHunterController.chooseTarget`). Per-attempt feedback escalates by `feedbackStage`: a start warning toast, a "flare" danger toast at the half-way surge (`SHRINE_FLARE_AT`), a grove-wide success on completion; the ladder resets when an abandoned ritual fully lapses.
- **Escape wins on EITHER trigger** (`checkEscapes`): reaching the exit pad (`ESCAPE_RADIUS_SQ`) OR simply CROSSING the gate (`pos.z <= GATE.z - 2`). The gate-cross is the robust trigger - once through the open archway a survivor has escaped even if the distant exit pad is hard to reach in the dark. The whole win chain (shrine lit, all-shrines/gate-open, escaped, nearest-to-exit) logs at INFO under a `[Kweebec][win]` tag. The gate is an open glowing archway (light blocks framing an opening, no blocking wall), not a door.
- **Survivor game mode is normalized to Adventure ONCE on entry, never per-tick.** `lazyPlayerSetup` sets Adventure only while `!st.isGameModeApplied()` (the hunter's marked target only holds on an Adventure survivor). Re-asserting it every tick fought an admin switching to Creative (it snapped back within ~1s - the 0.2.0 bug); [`../../hunter/AiHunterController`](../../hunter/AiHunterController.java) no longer touches game mode either. Do NOT re-add a per-tick game-mode force - an admin in Creative simply self-excludes from the hunter lock.
- **[`ChaseState`](ChaseState.java) holds shrines + the 0..1 corruption meter.** `corruptionTier()` (0/1/2) drives the heartbeat cadence + darkness depth; `hunterSpeed(rules)` maps corruption to a speed multiplier (best-effort - no live speed setter, see the hunter router). [`ShrineState`](ShrineState.java) is one shrine's anchor + lit/progress/channeller. Shrine relight + rescue use `ArenaLayout.INTERACT_RADIUS_SQ`; escape uses `ESCAPE_RADIUS_SQ`.
- **Outcome resolution** (`resolveOutcome`): any escaped = `ESCAPED`; cap reached = `TIMED_OUT`; everyone present cocooned = `CAUGHT`; the party fully empties = `ABORTED`; else `null` (keep running). Death/cocoon itself is handled in [`../../death/`](../../death/CLAUDE.md), not here.
