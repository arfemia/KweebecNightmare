# hunter/ - the pursuer seam (AI now, human-driven post-jam)

Router for `hunter/`. ALL methods run on the instance world thread (the state machine hops first). The relentless-chase mechanic is the trickiest verified Hytale seam in this mod - see the kweebec techspec (linked from the mod root).

- **[`HunterController`](HunterController.java) is the seam** so the round engine drives a pursuer without caring AI vs human (the asymmetric "play as the Kweebec" mode is architected-for, deferred): `spawn` / `tick` (corruption-speed ramp + override-only aggro) / `onAlert` (gate hard-lock) / `despawnAll` / **`hunterPositions(store)`** (0.3.0; the live hunter world positions the `ScareDirector` reads for the per-survivor proximity vignette). [`AiHunterController`](AiHunterController.java) is the only implementation.
- **0.3.0 - weighted ARCHETYPE ROSTER (no longer one role).** `AiHunterController` reads [`../asset/HunterArchetypeConfig`](../asset/HunterArchetypeConfig.java)`.all()` (generic, no hardcoded ids): `spawn` builds a weighted roster gated by each archetype's `SpawnTier` vs the current corruption tier + partySize, honoring `Count`/`Weight`; `applySpeed` now tracks **per-hunter** band state (was a single shared scalar) so each archetype paces on its OWN `SpeedBands`/`BandEffectIds`. The `AiHunterController(String)` ctor is unchanged (the fallback role) so `RoundService` is untouched. The 3 new archetypes (`KweebecNightmare_Lunger` Charge, `_Spitter` MaintainDistance kite, `_Ambusher` Teleport blink) are Variant roles of NEW sibling templates cloned from the Blight template - each REPLACES the Attack-state `Intelligent_Chase` motion with its special BodyMotion BUT retains the de-leashed Seek chase as the always-on fallback, so a rejected special motion degrades to a fast/tanky stalker rather than a broken role. **In-game gate:** that the de-leashed Attack state actually drives the ranged/teleport motion (not melee fallback); Teleport is Experimental.
- **The relentless chase is the ROLE's OWN Hostile AI, NOT a per-tick re-stamp** (the old `role.setMarkedTarget` re-stamp is GONE). The Blight Variant ([`KweebecNightmare_Blight.json`](../../../../../../resources/Server/NPC/Roles/KweebecNightmare/KweebecNightmare_Blight.json)) sets `ViewRange`/`AlertedRange`/`HearingRange` = 256 + `ViewSector` 360, so the role pursues the nearest survivor across the whole arena natively. The **soft leash was removed from the Template Attack state** ([`Template_KweebecNightmare_Blight.json`](../../../../../../resources/Server/NPC/Roles/KweebecNightmare/Templates/Template_KweebecNightmare_Blight.json)): it caused the Attack -> Idle -> re-Alerted infinite re-aggro loop, so the `Intelligent_Chase` exports were moved off `Idle` to `Search` and the hunter never gives up / returns to its den (the arena is bounded, so unbounded pursuit is safe).
- **Perfect Utils `AggroAPI` is an OVERRIDE-ONLY layer** (a HARD dependency, in the manifest `Dependencies`), used ONLY for criteria-based aggro switching the natural AI would not pick: the gate-alert lock + the loudest shrine channeller. `AiHunterController.tick` `taunt`s ONLY when `chooseTarget` differs from the natural nearest survivor, so a SOLO survivor is never taunted (no `AggroComponent` ever lands on them). `chooseTarget` ranks gate-alert > channeller > nearest; an override fires a ~2.5s `taunt` burst (`redirectAggro` hands the target to the combat AI, then expires), `clearTaunt` releases it guarded by `isTaunting`. **Gotcha:** an `AggroComponent` left on a player who DIES crashed teardown via Perfect Utils `removeComponent` (dying player's changed archetype no longer has it); fixed in the Perfect Utils repo ([`Developer-Utils`](../../../../../../../../../../hytale-clients/Developer-Utils), commit dc9524d) AND avoided here by the override-only design.
- **Spawns the pack hostile Kweebec role** (`KweebecNightmare_Blight`, a `Variant` of the pack template `Template_KweebecNightmare_Blight` - a parameterized clone of vanilla `Template_Kweebec_Razorleaf` that exposes `DefaultPlayerAttitude` as a Parameter so the Variant sets it `Hostile`; the vanilla template hardcodes `Neutral`, so a plain Variant-`Modify` of it failed validation) via `NPCPlugin.spawnEntity` at `ArenaLayout.HUNTER_DEN`; `onAlert` hard-locks every hunter onto the single nearest survivor to the gate (the next `tick` taunts them). **Corruption-scaled speed IS live** (not baked-only - the old "no runtime setter" read was wrong): each tick `applySpeed` snaps `ChaseState.hunterSpeed(ruleSet)` to a pre-authored `KweebecNightmare_HunterPace_*` EntityEffect and swaps it on every hunter via `EffectControllerComponent.addEffect`/`removeEffect` (mirroring `CocoonService`), and the engine folds that effect's `ApplicationEffects.HorizontalSpeedMultiplier` into the per-tick walk speed (`MotionControllerWalk.getMaximumSpeed` = baked `MaxSpeed` * the effect multiplier); the role `MaxSpeed` is the 1.0x baseline, the band assets live in the pack at `Server/Entity/Effects/KweebecNightmare/`. Other escalation: darkness + heartbeat tier + the alert lock-on. Full proof + the band ladder: the `kweebec-hunter-runtime-speed` research note in the parent repo.
- **Extra-spawn rules are ziggfreed-common's `EncounterRule` now** (migrated off kweebec's `SpawnRuleAsset`): `evaluateSpawnRules(..., SpawnTrigger, tierOrSeconds)` reads `KweebecNightmareAPI.resolveSpawnRules()` (-> `List<EncounterRuleAsset>` over `common.instance.encounter.EncounterRuleConfig` + the runtime override seam), gates each via the common `EncounterDirector`, and places via `SpawnPlacement`. The asset's `Trigger`/`Placement` are free STRINGS in common; kweebec's domain enums live HERE as [`SpawnTrigger`](SpawnTrigger.java) + [`SpawnPlacementKind`](SpawnPlacementKind.java) (with `fromString` parsing the common asset's getters). Content authors at `Server/ZiggfreedCommon/EncounterRules/`.
