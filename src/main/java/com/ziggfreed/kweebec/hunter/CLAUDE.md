# hunter/ - the pursuer seam (AI now, human-driven post-jam)

Router for `hunter/`. ALL methods run on the instance world thread (the state machine hops first). The marked-target mechanic is the trickiest verified Hytale seam in this mod - see the kweebec techspec (linked from the mod root).

- **[`HunterController`](HunterController.java) is the seam** so the round engine drives a pursuer without caring AI vs human (the asymmetric "play as the Kweebec" mode is architected-for, built post-jam): `spawn` / `tick` (re-assert marked target) / `onAlert` (gate hard-lock) / `despawnAll`. [`AiHunterController`](AiHunterController.java) is the only implementation today.
- **The marked target is re-asserted EVERY tick** (`role.setMarkedTarget(MarkedEntitySupport.DEFAULT_TARGET_SLOT, survivorRef)`) - the engine clears a marked PLAYER target each frame unless re-asserted, and only honors it on an **Adventure-mode** survivor. Survivors are put in Adventure once on entry by [`../mode/chase/ChaseMode`](../mode/chase/ChaseMode.java); this controller no longer forces game mode (the removed `ensureAdventure` fought a Creative admin every tick). An admin in Creative simply self-excludes from the lock - do NOT re-add a per-tick game-mode force here.
- **Spawns the pack hostile Kweebec role** (`KweebecNightmare_Blight`, a clone of `Template_Kweebec_Razorleaf`) via `NPCPlugin.spawnEntity` at `ArenaLayout.HUNTER_DEN`; the spawn callback + `tick` lock the nearest active survivor, and `onAlert` hard-locks every hunter onto the single nearest survivor to the gate. **No live walk-speed setter exists in 0.5.3** (speed is baked on the role's Walk controller) - corruption-scaled speed is an asset/post-jam concern; escalation today is darkness + heartbeat tier + the alert lock-on.
