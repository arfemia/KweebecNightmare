# npc/ - the Grove Warden guide spawner

Router for `npc/`. Kweebec's thin policy layer over the lifted `ziggfreed-common` NPC primitives ([`com.ziggfreed.common.npc`](../../../../../../../ziggfreed-common/src/main/java/com/ziggfreed/common/npc/CLAUDE.md): `NpcSpawnService`, `ActionOpenDialogue`/`ZigOpenDialogue`, `NpcDialogueDepsRegistry`, `NpcActions`). The spawn primitive + press-F action live in common; this package keeps only the kweebec-specific placement policy, exactly as the MMO's `QuestGiverSpawnService` layers on its primitive.

- **[`KweebecGuideSpawn`](KweebecGuideSpawn.java)** auto-spawns `KweebecNightmare_Guide` once per world on `PlayerReadyEvent` (registered in [`KweebecNightmarePlugin.setup`](../KweebecNightmarePlugin.java)), near the world spawn via `NpcSpawnService.resolveSpawnPosition` + a small offset, all inside `world.execute`. Idempotency is a kweebec-side per-world `Set` (NOT a lifted singleton placement store - that would clobber across mods); a per-world guide UUID is recorded via the spawn's post-spawn callback so `reposition` (the `/kweebec spawnguide` debug hatch) can despawn the old one first. Round instance worlds are entered by teleport (no `PlayerReadyEvent`), so the guide never spawns inside a round.
- **Press-F wiring is in the role asset, not here.** `Server/NPC/Roles/KweebecNightmare/KweebecNightmare_Guide.json` (a `Type: Generic` Passive role copied from the MMO's `MMO_QuestGiver`) carries the `InteractionInstruction` that fires `{ "Type": "ZigOpenDialogue", "Dialogue": "nightmares_intro", "ContextNpc": "guide" }`. The action + its `ZigOpenDialogue` registration + the `DialoguePageDeps` provider all live in common / [`../dialogue/KweebecDialogue`](../dialogue/KweebecDialogue.java); the plugin calls `NpcActions.register()` + `NpcDialogueDepsRegistry.set(KweebecDialogue::deps)` in `setup()` BEFORE the role asset loads, or `{ "Type": "ZigOpenDialogue" }` fails to parse.
